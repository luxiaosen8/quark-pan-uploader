# 夸克网盘批量上传分享工具设计文档

**日期：** 2026-03-07  
**目标平台：** Windows 本地桌面 GUI  
**技术方向：** Python + PySide6

## 1. 目标

构建一个 Windows 本地桌面工具，允许用户输入夸克网盘 `Cookie`，浏览网盘目录，选择本地源目录和目标网盘目录，然后将本地源目录下的所有**一级子文件夹**分别上传到目标网盘目录中对应的同名文件夹下，并在每个一级子文件夹上传完成后为其创建分享链接，将成功获取的分享链接按“一行一个链接”的格式记录到本地 `txt` 文件中。

## 2. 成功标准

- 用户可在 GUI 中输入并校验 `Cookie`
- 用户可刷新并查看网盘目录信息
- 用户可选择本地目录并预览一级子文件夹
- 用户可在网盘目录树中选择目标目录
- 程序可按一级子文件夹为单位递归上传内容
- 程序可在每个一级子文件夹上传完成后发起分享并获取最终链接
- 程序可将链接实时追加写入本地 `txt` 文件
- 程序可实时输出日志、进度、成功/失败状态

## 3. 范围

### 3.1 首版包含

- 单账号 `Cookie` 认证
- 网盘目录刷新与选择
- 本地目录扫描（仅一级子文件夹）
- 远端同名目录创建 / 复用
- 文件递归上传
- 文件夹分享创建与分享链接获取
- `txt` 结果文件与明细结果文件输出
- 实时日志、状态表格、停止任务

### 3.2 首版不包含

- 多账号轮换
- OAuth / Token 登录
- 浏览器自动抓 Cookie
- 分片级真正断点续传恢复
- 自动跳过远端同名文件的强一致去重策略
- 高级分享策略编辑器

## 4. 关键业务规则

### 4.1 本地目录规则

- 只处理用户所选本地根目录下的**一级子文件夹**
- 根目录下的散文件不参与上传
- 空子文件夹默认跳过并记录日志
- 每个一级子文件夹内部内容递归上传

### 4.2 远端目录规则

- 用户先选中一个网盘目标目录
- 对于每个一级子文件夹，在目标目录下创建或复用同名远端目录
- 子文件夹内部的相对路径结构在远端保持一致

### 4.3 分享规则

- 分享对象为“上传完成后的远端文件夹”
- 目标策略为“公开分享、无提取码、尽量请求长期有效”
- 最终是否真正满足“永久有效 / 无提取码”以接口实际返回为准
- 若接口拒绝策略，不做静默降级，直接报错并记录

### 4.4 输出规则

- 主结果文件：`share_links.txt`，一行一个分享链接
- 明细结果文件：记录本地目录、远端目录、分享 ID、分享链接、状态、错误信息
- 成功获取链接后立即追加写入主结果文件

## 5. 总体架构

系统分为四层：

1. **GUI 层**：负责用户输入、目录选择、表格展示、日志展示、任务控制
2. **任务编排层**：负责扫描目录、调度上传、状态管理、停止/重试、结果落盘
3. **夸克 API 客户端层**：负责目录、上传、分享、任务查询等 HTTP 封装
4. **本地持久化层**：负责配置、日志、结果文件写入

## 6. 模块设计

### 6.1 GUI 层

- `MainWindow`
  - Cookie 输入
  - 刷新网盘按钮
  - 本地目录选择
  - 远端目录树
  - 任务控制按钮
  - 进度条
  - 子文件夹任务表格
  - 日志面板

### 6.2 任务编排层

- `UploadShareCoordinator`
  - 扫描一级子文件夹
  - 逐个子文件夹推进任务
  - 负责状态流转
  - 负责停止标记与错误处理
- `FolderUploadJob`
  - 表示单个一级子文件夹任务

### 6.3 API 客户端层

- `QuarkSession`
  - 统一请求头、Cookie、重试、超时、错误封装
- `QuarkUserApi`
  - 用户信息 / 容量 / Cookie 校验
- `QuarkFileApi`
  - 列目录 / 创建目录 / 查询任务 / 文件信息
- `QuarkUploadApi`
  - 预上传 / 上传 / 完成上传
- `QuarkShareApi`
  - 创建分享 / 查询分享详情

### 6.4 本地持久化层

- `AppSettings`
  - 保存 UI 偏好、Cookie 保存开关、默认输出目录等
- `ResultWriter`
  - 追加写入 `txt` / `csv`
- `StructuredLogger`
  - 写入 `jsonl` 日志

## 7. 核心流程

### 7.1 连接与刷新

1. 用户输入 Cookie
2. 点击“刷新网盘”
3. 调用轻量接口进行 Cookie 有效性校验
4. 加载用户信息与网盘目录树
5. 更新界面状态为“已连接”

### 7.2 本地扫描

1. 用户选择本地目录
2. 程序枚举一级子文件夹
3. 统计每个子文件夹的文件数、总大小
4. 在表格中显示待处理项目

### 7.3 上传与分享

对每个一级子文件夹：

1. 在目标网盘目录下创建或复用同名目录
2. 递归创建必要的子目录结构
3. 遍历文件清单并执行上传
4. 子文件夹全部文件完成后发起分享
5. 查询并整理最终分享链接
6. 写入结果文件并更新表格状态

## 8. 上传链路设计

由于公开资料显示夸克上传更接近“预上传 -> 对象存储上传 -> 完成上传”的多段流程，因此上传模块必须与 GUI 解耦，并以状态机方式组织：

1. 预上传：提交文件元数据与目标目录信息
2. 秒传判断：如果后端命中秒传，则跳过数据上传
3. 数据上传：普通上传或分片上传
4. 完成上传：提交确认或轮询异步任务
5. 返回远端文件信息

## 9. 异常处理策略

- Cookie 失效：阻止开始任务并提示用户重新刷新
- 目录加载失败：允许重试，不清空本地目录选择
- 文件读权限错误：跳过单文件并记录错误
- 上传失败：按文件或分片粒度重试，超过阈值后标记当前子文件夹失败
- 分享失败：该子文件夹不写主结果文件，错误写入明细文件和日志
- 写文件失败：弹窗提示，并保留内存结果供手动导出
- 限流 / 风控：指数退避，多次失败后暂停队列

## 10. GUI 交互设计

## 11. 输出文件设计

### 11.1 主结果文件

- 路径：`output/share_links.txt`
- 格式：纯文本，一行一个链接

### 11.2 明细结果文件

- 路径：`output/share_results.csv`
- 字段：
  - `local_folder_name`
  - `local_folder_path`
  - `remote_folder_name`
  - `remote_folder_fid`
  - `share_id`
  - `share_url`
  - `status`
  - `created_at`
  - `error_message`

### 11.3 结构化日志

- 路径：`output/logs/YYYY-MM-DD.jsonl`
- 每条记录包含：时间、级别、阶段、子文件夹名、消息、扩展字段

## 12. 技术选型

- `Python 3.12+`
- `PySide6`：桌面 GUI
- `requests`：HTTP 请求
- `pytest`：单元测试
- `pytest-qt`：GUI 相关测试
- `pydantic`：配置与数据模型校验
- `pyinstaller`：Windows 打包

## 13. 安全与维护性

- Cookie 默认仅保存在本地配置文件中，提供“不保存”选项
- 敏感信息写日志时脱敏
- HTTP 原始响应仅在调试模式下落盘
- 将上传、分享策略集中到配置对象中，避免散落常量难以维护

## 14. 风险与假设

### 14.1 风险

- 夸克接口属于非官方逆向接口，字段与行为可能随时变化
- 分享策略不一定始终支持“无提取码 / 长期有效”
- 大文件上传可能涉及额外签名或对象存储参数

### 14.2 假设

- 用户能提供当前有效的 PC 端 Cookie
- GUI 运行环境为 Windows 10/11
- 目标账号具备正常上传与分享权限

## 15. 结论

首版以“单账号、一级子文件夹批量上传、逐个文件夹分享、实时写链接”为核心，采用 `Python + PySide6` 的单机架构。通过将 GUI、任务编排、夸克 API、持久化分层隔离，可以在后续接口变化时最小化影响面，并为未来的多账号、断点续传和高级分享策略预留扩展空间。
