# 夸克网盘上传器标签页交互联动优化设计

**日期：** 2026-03-09  
**范围：** 仅调整 `src/quark_uploader/gui/main_window.py` 的标签页交互联动，并补充对应 GUI 测试与文档。

## 1. 目标

在已经完成的“上栏操作区 + 下栏标签页工作台”布局基础上，继续优化操作流，让界面在执行任务时更顺手，同时保持“操作区优先”的交互原则。

## 2. 本次行为约定

- 应用启动后默认展示 `上传任务` 标签页
- 当用户从其他标签页点击 `开始上传` 时，界面应自动切回 `上传任务` 标签页
- 写入日志时不自动切到 `运行日志` 标签页
- 更新当前远端目录摘要时不自动切到 `目标网盘目录` 标签页

## 3. 方案选择

### 方案 A：在 `MainWindow` 内直接处理按钮到标签页的联动（推荐）
- 由 `start_button.clicked` 直接触发切换到任务页
- 优点：改动最小，局限在 UI 层，不污染 controller 业务流程
- 缺点：交互联动逻辑分散在视图内部，但本次规模很小，可接受

### 方案 B：由 controller 在 `start_upload()` 中主动切换标签页
- 优点：所有交互流集中在 controller
- 缺点：需要改动已有 controller，且当前工作区存在未提交相关改动，风险更高

## 4. 决策

采用 **方案 A**。

原因：本次仅需增加一个轻量的 UI 联动行为，保持在 `MainWindow` 内即可，既符合“表现层改动优先”的约束，也能避免覆盖当前工作区里 controller 的其他未提交变更。

## 5. 测试策略

- 先写失败测试：当当前标签页不是 `上传任务` 时，点击 `开始上传` 后应切回任务页
- 再实现最小代码：为 `MainWindow` 暴露任务页切换方法，并将 `start_button.clicked` 绑定到该方法
- 最后跑 GUI 目标测试与全量测试
