# 上传模式按钮联动收尾设计文档

**日期：** 2026-03-09  
**范围：** 仅收敛 `controller.py` 与相关 GUI 测试中残留的上传模式切换引用，保持与当前 `main_window.py` 的按钮化实现一致。

## 1. 背景

主界面的上传模式切换控件已经从单选按钮改为互斥的可勾选按钮：

- `upload_mode_batch_button`
- `upload_mode_single_button`

但 controller 与部分测试文件仍残留旧的 radio 引用，需要同步收尾，否则仓库状态无法整理干净。

## 2. 方案

- controller 改为监听两个模式按钮的点击事件
- 保留 `_set_upload_mode()` 作为唯一模式切换入口
- 删除旧的 radio toggle 回调
- 同步更新 GUI 测试中的控件引用

## 3. 风险控制

- 不改上传逻辑
- 不改设置逻辑
- 不改主窗口布局
- 仅做控件引用与事件连接层面的同步

## 4. 成功标准

- 工作区不再残留这 3 个按钮切换相关源码改动
- 全量测试通过
- 上传模式切换行为与当前主界面实现一致
