# 夸克网盘批量上传分享工具 UI 优化设计文档

**日期：** 2026-03-08  
**方向：** 专业桌面工具风  
**范围：** `main_window.py` 为主，`controller.py` 小范围联动，禁止改动上传/分享业务逻辑

## 1. 目标

将当前功能完整但偏原型化的 PySide6 界面优化为更接近成熟桌面工具的表现：

- 分区清晰
- 主次明确
- 状态可快速感知
- 长时间使用更易读
- 保持现有业务流程和测试基础不被破坏

## 2. 设计原则

1. **只优化表现层，不改业务语义**
2. **继续使用现有 Widgets 技术栈，不引入 QML 或新框架**
3. **优先重组布局，再做样式增强**
4. **复用现有控件实例，避免 controller 大改**
5. **通过测试保证交互不回退**

## 3. 布局方案

采用“顶部摘要 + 中区双栏 + 任务区 + 底部日志”的专业桌面工具布局：

- 顶部摘要卡片
  - 标题
  - 连接状态
  - 账号信息
  - 当前动作
  - 任务摘要
  - 总进度条
- 中部双栏
  - 左侧：连接、本地目录、操作按钮
  - 右侧：目标网盘目录树 + 当前选择提示
- 下方主工作区
  - 上传任务表格
- 底部日志卡片
  - 运行日志

## 4. 视觉语言

- 浅色背景 + 白色卡片
- 蓝色主按钮
- 中性灰边框与留白
- 表格、目录树、日志区统一风格
- 状态值以颜色和层级强化可读性

## 5. 边界与非目标

### 本次包含
- 重新组织主窗口布局
- 增加卡片式分区
- 强化状态摘要区
- 优化任务表格、日志区和目录树的可读性
- 小范围补充“当前选择目录”展示

### 本次不包含
- 上传逻辑变更
- 分享逻辑变更
- 输出文件逻辑变更
- 设置逻辑变更
- 深色模式切换
- 多主题系统
- 动画系统

## 6. 测试策略

- 先补/改 UI 结构测试
- 再做布局和样式实现
- 跑 GUI tests 和全量 tests
- 不做像素级截图对比测试，只验证结构、文案、关键展示和交互不回退

## 7. 成功标准

- 页面明显更像专业桌面工具
- 连接、本地选择、远端目录、任务、日志分区明确
- 表格和日志更易读
- 现有控制器绑定与功能测试保持通过
- 不改动与 UI 表现无关的业务代码
