# 单目标上传模式设计文档

**日期：** 2026-03-08  
**目标：** 在保留现有批量子文件夹上传模式的前提下，新增“单文件/单文件夹”上传模式。

## 1. 背景

当前工具只支持：

- 选择一个本地根目录
- 扫描一级子文件夹
- 逐个上传一级子文件夹并分享远端文件夹

用户新增需求：

- 支持直接选择单个文件夹作为一个任务上传
- 支持直接选择单个文件作为一个任务上传
- 单文件夹任务递归上传内部内容，并分享远端文件夹
- 单文件任务直接上传文件本身，并分享该远端文件
- 状态、日志、结果写入和回调语义应尽量与现有模式保持一致

## 2. 设计原则

1. **保留批量模式不变**
2. **新增单目标模式，而不是隐式改变旧逻辑**
3. **通过扩展任务模型兼容 file / folder 两种任务**
4. **尽量复用现有上传执行链路**
5. **输出文件和状态流转保持一致，仅增加必要的类型字段**

## 3. 功能设计

### 3.1 上传模式

新增两种上传模式：

- `batch_subfolders`：保留现有一级子文件夹批量上传
- `single_target`：新增单文件/单文件夹上传

### 3.2 单目标模式行为

#### 选择单个文件夹
- 文件夹本身作为一个任务
- 远端保留该文件夹名
- 递归上传内部内容
- 分享远端文件夹

#### 选择单个文件
- 文件本身作为一个任务
- 直接上传到当前选中的远端目标目录
- 分享远端文件本身

### 3.3 本次范围限制

本次仅支持：

- 一次选择 **一个文件夹**
- 或一次选择 **一个文件**

不包含：

- 多文件多选
- 多文件夹多选
- 文件与文件夹混选

## 4. 模型设计

建议新增：

- `UploadMode`：`batch_subfolders` / `single_target`
- `TaskSourceType`：`folder` / `file`

并扩展任务模型，使批量子文件夹任务、单文件夹任务、单文件任务都能统一进入任务表格、上传计划和执行器。

## 5. 执行链路设计

### 5.1 目录任务

适用于：
- 批量模式下的一级子文件夹
- 单目标模式下选中的文件夹

行为：
- 递归生成文件清单
- 规划并创建远端目录结构
- 上传后分享远端文件夹

### 5.2 文件任务

适用于：
- 单目标模式下选中的单个文件

行为：
- 只生成一个文件清单项
- 不生成递归远端目录要求
- 文件直接上传到所选远端目标目录
- 上传后直接分享该远端文件

## 6. GUI 设计

在操作区新增一个轻量模式切换：

- 批量子文件夹
- 单文件/单文件夹

单目标模式下显示：

- 选择单个文件夹
- 选择单个文件

任务表格第一列标题改为更通用的：

- `任务名称`

## 7. 结果文件设计

保留现有：

- `share_links.txt`
- `share_results.csv`
- `share_results.jsonl`
- `events.jsonl`

新增兼容字段：

- `remote_item_type = folder / file`

## 8. 风险控制

### 需要特别注意
1. 旧批量模式不能回退
2. 单文件任务不能错误走目录递归规划
3. 分享层不能再默认只面向文件夹语义
4. GUI 的开始上传条件应改为“已选上传来源”，而不是只认本地根目录

## 9. 成功标准

- 批量模式保持原有行为
- 单文件夹模式可递归上传并分享远端文件夹
- 单文件模式可上传并分享远端文件
- 状态、日志、结果写入与现有模式一致
- GUI 和全量测试通过
