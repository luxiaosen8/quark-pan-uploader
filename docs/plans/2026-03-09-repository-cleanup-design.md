# 仓库上传前清理设计文档

**日期：** 2026-03-09  
**范围：** 仅清理仓库中的本地调试残留、构建产物、缓存目录与忽略规则，不处理现有未提交源码改动。

## 1. 目标

为上传远程仓库做准备，清理与项目源码无关的本地杂物，使仓库目录结构更干净，并降低误提交本地产物的风险。

## 2. 本次清理范围

### 包含
- 删除本地构建产物：`build/`、`dist/`
- 删除本地运行输出：`output/`、`.local/`、`.pytest_cache/`
- 删除所有 `__pycache__/`
- 删除调试目录：`debug_large/`、`debug_small/`、`debug_stop/`
- 删除空测试输出目录：`cleanup-output/`、`custom-output/`
- 更新 `.gitignore`，防止调试目录再次出现在工作区

### 不包含
- 不删除 `.venv/`
- 不处理已有未提交源码改动
- 不修改业务代码与测试逻辑

## 3. 风险控制

- 仅删除可明确判定为缓存、构建产物、调试残留的目录
- 现有源码改动保持原样，不纳入本次提交
- 通过 `git status` 复核清理后工作区状态

## 4. 成功标准

- `debug_*` 目录从工作区消失且被忽略
- 本地构建/缓存目录已清理
- `.gitignore` 覆盖这些本地残留
- 清理提交不包含任何与本次需求无关的源码改动
