# 夸克网盘上传器操作区优先布局设计文档

**日期：** 2026-03-09  
**方向：** 操作区优先 + 下栏标签页工作台  
**范围：** 仅调整 `src/quark_uploader/gui/main_window.py` 的界面组织，并同步更新 `tests/gui/test_main_window.py` 与项目文档规则。

## 1. 目标

解决当前主界面“操作区占比过小、主次颠倒”的交互问题，将操作区升级为默认视线焦点与主要工作台。

本次优化后的主结构：

- 上栏：全宽操作区
- 下栏：标签页工作区
  - 上传任务（默认）
  - 目标网盘目录
  - 运行日志

## 2. 设计原则

1. **只改表现层，不改上传/分享业务逻辑**
2. **保留现有控件实例与控制器绑定**
3. **操作优先，信息查看次级**
4. **默认窗口高度下不应依赖滚动完成主要操作**
5. **仅在极端高度或 DPI 缩放下允许滚动承接**

## 3. 布局方案

### 3.1 上栏：操作区主工作台

操作区改为全宽卡片，内部包含两层内容：

- 顶部状态摘要条
  - 标题
  - 连接状态
  - 账号信息
  - 当前动作
  - 任务进度
  - 总进度条
  - 当前选中远端目录摘要
- 底部三分栏操作面板
  - 账号连接
  - 上传来源
  - 任务执行

### 3.2 下栏：标签页工作区

将原来的远端目录、任务表格、日志区收拢为 `QTabWidget`：

1. `上传任务` —— 默认页，承载任务表格
2. `目标网盘目录` —— 承载远端目录树
3. `运行日志` —— 承载日志面板

## 4. 交互约定

- 启动后默认展示 `上传任务` 标签页
- 不自动抢切到日志页
- 当前远端目录摘要始终显示在上栏操作区，避免必须切换标签页才能确认目标位置

## 5. 非目标

本次不包含：

- 上传逻辑调整
- 远端目录加载逻辑调整
- 控制器行为改写
- 主题系统或深色模式
- 新增复杂动画

## 6. 测试策略

- 先修改 GUI 结构测试，明确新布局约束
- 验证根布局为“上方操作区 + 下方标签页”
- 验证标签页顺序与默认页
- 验证默认窗口尺寸下操作区滚动条不会明显出现
- 最后执行全量测试，确保 UI 改造不影响现有行为
